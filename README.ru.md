## Зачем NOUZ?

Ты пишешь в Obsidian. Много. Со временем хранилище превращается в свалку — сотни заметок, связанных непонятно как, без системы и логики.

NOUZ решает эту проблему. Он читает твои заметки, анализирует содержание и **сам выстраивает граф знаний** — куда что относится, какой у каждой заметки «знак», где семантические связи между разными ветками.

Никаких API-ключей. NOUZ работает с твоей собственной моделью эмбеддингов.

---

## Что умеет NOUZ

### Строит граф

Добавь в заметку `parents:` — и NOUZ автоматически поместит её в иерархию. Никакой ручной сортировки по папкам.

### Классифицирует по содержанию

С помощью эмбеддингов (локальная модель) NOUZ понимает, о чём заметка, и присваивает ей «знак» — T (технологии), S (наука), H (гуманитарное) или любой другой, какой задашь сам.

### Находит неочевидные связи

Заметки из разных веток графа могут быть семантически близки. NOUZ находит эти мосты и предлагает связать их.

### Следит за дрейфом знаний

Со временем содержимое ветки меняется. NOUZ показывает, когда реальный состав заметок расходится с заявленной темой — это `core_drift`, сигнал что ветка выросла за пределы своего домена.

## Три режима для разных задач

| Режим | Что делает | Эмбеддинги? |
|-------|------------|-------------|
| **LUCA** | Чистый граф — только ссылки и иерархия | ❌ |
| **PRIZMA** | Полная семантика — классификация, мосты, дрейф | ✅ |
| **SLOI** | Строгая 5-уровневая иерархия с контролем | ✅ |

Начни с **LUCA** — просто подключи и добавь ссылки. Переходи на PRIZMA или SLOI, когда захочешь семантику.

## Как это работает

### 1. Пишешь заметку

```yaml
---
type: module
level: 3
sign: T
parents:
  - Machine Learning
---

Твоя заметка here.
```

### 2. NOUZ строит граф

- Читает YAML → строит DAG (направленный ациклический граф)
- Векторизует текст → вычисляет близость к твоим ядрам
- Предлагает знак, уровень, родителей
- Находит мосты к другим веткам

### 3. Структура рождается из содержания

```
T (core)
 ├── TH (pattern: AI)
 │   ├── H (module: ML)
 │   │   ├── SH (quant: neural-networks.md) — T
 │   │   └── S (quant: transformers.md) — T
 │   └── H (module: Ethics)
 │       └── TH (quant: ai-safety.md) — H
 └── S (pattern: Physics)
     └── ...
```

Чем больше пишешь - тем умнее становится и твоя база знаний, и агент, работающий с Nouz.

## Как работает классификация

### Ядра — координатные оси

В `config.yaml` ты определяешь 2–5 доменов как текстовые описания. Сервер превращает их в векторы-эталоны — это **оси координат** в многомерном пространстве эмбеддингов.

```yaml
mode: prizma

etalons:
  - sign: T
    name: Technology
    text: "programming software architecture machine learning neural networks"
  - sign: S
    name: Science
    text: "physics chemistry biology mathematics formal logic theorems"
  - sign: H
    name: Humanities
    text: "philosophy psychology sociology history literature ethics"
```

**Про качество ядер:** важна не точность описания, а разделённость между ядрами. После записи — запусти `calibrate_cores` и проверь `pairwise_cosine`. Значение выше 0.55 между любыми двумя ядрами говорит о том, что они слишком похожи и классификатор будет путать домены. Это аналог ортогональных базисных векторов: чем ортогональнее — тем точнее проекция.

### Присвоение знака

Для каждой заметки вектор её содержимого сравнивается со всеми эталонами:

```
scores     = {S: cosine(note, core_S), T: cosine(note, core_T), H: cosine(note, core_H)}
spread     = max(scores) - min(scores)
```

Если `spread < 0.05` — заметка равноудалена от всех ядер, знак не определяется. Это не ошибка: заметка действительно принадлежит нескольким доменам или её содержимого недостаточно для классификации.

```
adjusted   = {k: scores[k] - min(scores) for k in scores}
percent    = {k: adjusted[k] / sum(adjusted) * 100 for k in adjusted}
sign       = все домены, где percent[k] >= 30%
```

Если два домена набирают ≥ 30% — знак составной: `TS`, `SH`. Это не противоречие, а спектр: заметка живёт на границе между доменами.

### Наследование знака

Знак течёт сверху вниз по иерархическим связям:

```
L1 ядро      ← задаётся вручную, не меняется
L2 паттерн   ← задаётся вручную + эмбеддинг добавляет второй знак (если ≥ 30%)
L3 модуль    ← наследует знак от родителя-паттерна
L4 квант     ← гибрид: знак L3 + эмбеддинг контента
L5 артефакт  ← наследует знак кванта-родителя
```

Приоритет: `sign_manual (YAML) > sign_auto (эмбеддинг) > унаследованный`

Ручной знак в YAML **никогда не перезаписывается** автоматикой.

### Уверенность знака: auto vs weak_auto

Spread-нормализация отвечает на вопрос «какое ядро *относительно ближе*». Но это не отвечает на вопрос «насколько заметка вообще близка к этому ядру в абсолютном смысле».

Для этого есть `confident_cosine` — порог на `max(cosine)`:

```
max_cosine >= confident_cosine → sign_source = "auto"      (знак надёжный)
max_cosine < confident_cosine  → sign_source = "weak_auto" (знак относительный)
```

**Что это означает на практике:**

`weak_auto` возникает когда у заметки есть относительный победитель среди ядер (spread нормальный), но в абсолюте все косинусы низкие. Это происходит когда:
- Заметка короткая или написана не на языке эталонов
- Эталоны плохо покрывают тему заметки
- Модель эмбеддингов не сильна в этом домене

**Влияние на семантические мосты:**

Вот где это становится важным. Семантические мосты предлагаются только между заметками с *разными* знаками — предполагается что если знаки совпадают, это уже «одна область». Но если знак `weak_auto` — это предположение ненадёжно.

```
sign_source = "auto"      → знак считается закрытым. Мосты к тому же ядру не предлагаются.
sign_source = "weak_auto" → знак открытый. Мосты к заметкам с тем же ядром всё равно предлагаются.
```

Это даёт тебе выбор: либо поставить знак вручную (закрыть домен), либо позволить системе продолжать предлагать связи из всех направлений.

**Настройка порога:**

```yaml
thresholds:
  confident_cosine: 0.6  # для большинства моделей (e5, BGE, multilingual)
  # confident_cosine: 0.75  # для nomic-embed (высокий baseline 0.74–0.83)
```

Чем выше порог — тем строже условие уверенности, тем больше заметок получают `weak_auto`. Калибруй под свою модель.

---

### core_mix — реальность снизу вверх

Знак течёт сверху вниз (намерение). `core_mix` течёт снизу вверх (реальность):

```
Квант (L4) → обновляет core_mix родителя-модуля (L3)
Модуль (L3) → агрегирует core_mix родителя-паттерна (L2)
```

Каждый модуль накапливает усреднённое распределение доменов всех своих квантов. Когда задекларированный знак (намерение) расходится с `core_mix` (реальность) — система сообщает о `core_drift`.

Это не ошибка. Это информация о том, как база знаний эволюционировала. Ветка с `sign=T` может постепенно накопить 60% заметок по математике — и `core_drift` это покажет.

Такая двусторонняя структура потока — ограничения сверху вниз и свидетельства снизу вверх — напоминает архитектуру иерархических предсказательных систем: верхние уровни задают ожидание, нижние возвращают расхождение с ним.

## Семантические мосты

`suggest_metadata` возвращает кандидатов на кросс-доменные связи. Все мосты помечены `proposed: true` и требуют явного подтверждения.

**Алгоритм:**

```
Для заметки A (sign=S) и заметки B (sign=T):
  если cosine(embed(A), embed(B)) >= 0.55 → предложить связь
```

Находит заметки, которые **семантически схожи в целом**, но принадлежат разным доменам. Заметка про термодинамическую энтропию и заметка про сжатие данных могут иметь высокое сходство эмбеддингов — обе про эффективное кодирование информации. Мост обнаруживает, что они говорят об одном с разных сторон.

Порог 0.55 настраивается в `config.yaml` через `semantic_bridge_threshold`.

## Настраивай под себя

```yaml
mode: prizma

etalons:
  - sign: T
    name: Technology & Engineering
    text: "programming software architecture infrastructure machine learning neural networks
           algorithms frameworks database cloud computing"
  - sign: S
    name: Science & Mathematics
    text: "physics chemistry biology mathematics formal logic theorems cosmology quantum
           mechanics research methodology"
  - sign: H
    name: Humanities & Arts
    text: "philosophy psychology sociology history literature art culture ethics
           cognitive science epistemology linguistics"

thresholds:
  sign_spread: 0.05           # минимальный разброс для классификации
  pattern_second_sign_threshold: 30.0  # порог второго знака (%)
  semantic_bridge_threshold: 0.55      # порог семантического моста
```

## Инструменты

### Все режимы

| Инструмент | Описание |
|-----------|----------|
| `read_file` | Прочитать заметку с YAML-метаданными |
| `write_file` | Создать или обновить заметку |
| `list_files` | Список с фильтрами по level, sign, подпапке |
| `get_children` | Обход DAG вниз (все дети) |
| `get_parents` | Обход DAG вверх (родители) |
| `index_all` | Переиндексировать всё хранилище |
| `format_entity_compact` | Формула сущности: `(дети)[сущность]{родители}` |

### PRIZMA / SLOI

| Инструмент | Описание |
|-----------|----------|
| `calibrate_cores` | Векторизовать тексты ядер → эталонные векторы |
| `recalc_signs` | Переклассифицировать все заметки по эмбеддингам |
| `recalc_core_mix` | Пересчитать bottom-up агрегацию доменов |
| `suggest_metadata` | Предложить знак, уровень, семантические мосты |
| `suggest_parents` | Найти родителей по семантическому сходству |
| `embed` | Получить вектор эмбеддинга для произвольного текста |

## Приватность: что остаётся локальным

| Компонент                                          | Локально?               |
| -------------------------------------------------- | ----------------------- |
| **Эмбеддинги** (LM Studio / Ollama)                | ✅ Да                    |
| **Твои заметки** (сырые файлы)                     | ✅ Да                    |
| **NOUZ сервер**                                    | ✅ Да                    |
| **Контекст AI-агента** (что видит облачная модель) | ❌ Нет — уходит в облако |

NOUZ работает локально. Но когда подключаешь облачного AI-агента (Claude, ChatGPT и т.д.), контент, который он видит — включая твои заметки — уходит в облако этого провайдера. Это вне контроля NOUZ.

**Твой выбор:** используй локальных агентов или принимай компромисс.
